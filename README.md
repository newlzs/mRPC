> RPC: 提供RPC服务   
  client： 调用者   
  server： 被调用者  
  share： client 和 server共享的数据

![img](https://s3.51cto.com/oss/201906/17/fd5b5686336b0a1212398d8ea8fe6f66.jpg-wh_651x-s_3461264051.jpg)  

## RPC架构
1. 发布服务 （服务端）  
    将服务发布出去, 并注册到注册中心
    
2. 服务中心（服务端）  
    记录已经发布的服务，接口名： 服务
    本地实现： 通过 本地服务端的map记录  
    远程实现： 通过网络操作远程的注册中心。
        
    
3. 服务消费者（客户端）  
    通过动态代理,（接口名+方法名）获取远程服务地址然后连接调用远程服务
