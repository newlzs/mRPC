> RPC: 提供RPC服务   
  client： 调用者   
  server： 被调用者  
  share： client 和 server共享的数据

![img](images/rpc.jpg)
## RPC架构
1. 发布服务 （服务端）  
    将服务发布出去, 并注册到注册中心   
    本地实现： 通过本地服务端的map记录   
    远程实现： 通过网络操作远程的注册中心。 
    
2. 注册中心（服务端，客户端）   
    Nacos实现   
    服务端向其注册/删除服务  
    客户端根据服务名称获取服务地址  
        
    
3. 服务消费者（客户端）  
    通过动态代理,（接口名+方法名）获取远程服务地址然后连接调用远程服务
